CommTest

This application facilitates testing of the CRSocket component, using default settings. It consists of a controller/responder app pair that are identical, except that they are initiated with "controller" and "responder" clientRoles respectively.

Note that the accessToken must be a non-empty string of Ascii printable characters.

Functionality

The primary purpose of the app is to monitor:

  • Number of events transmitted between eventreceiver resources and requesting clients.
  • Number of events transmitted between CRSocket objects.
  • Current state and subState of the CRSocket object, and its internal CREventReceiver and CREventSender objects.
  • Success rate, delays and reception accuracy of crEvents transmitted between CRSocket clients.
  • Handling of various types of errors and exceptions. This includes the pairing procedure. partnerClientId assignment can only be done by the controller app.

For multiple events use event type "multi" and specify number of events, and min/max secs between events. The parameters will be set to the current event number and the current delay to the next event in millisecs. For single events you may optionally specify subType and two event parameters. In both cases it is possible to toggle the log, buffered and interceptable event flags.

The app displays:

  • Number of messageEvents transmitted to/from the eventsource server resource.
  • Amount of time CREventReceiver objects spends in connecting, open and closed states.
  • The round trip time of alive messsageEvents.
  • No of crEvents transmitted to/from partner clients.
  • The current state/substate of the crSocket, and it's internal CREventReceiver and CREventSender objects.
  • Conficuration parameters assigned to the crSocket: clientId, accessToken and partnerClientId.

It is also possible to monitor state changes and internal events in the browser JavaScript console.
