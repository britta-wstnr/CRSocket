CommTest

This application facilitates testing of the CRSocket component, using default settings. It consists of a controller/responder app pair that are identical, except that they are initiated with "controller" and "responder" clientRoles respectively.

Note that the default authToken is a non-empty string of Ascii printable characters.

Functionality

The primary purpose of the app is to monitor:

  • State changes, server communication and crSocket interaction under various circumstances.
  • Success rate, delays and reception accuracy of crEvents transmitted between clients.

The app displays:

  • Number of messageEvents transmitted to/from the eventsource server process.
  • Amount of time the eventReceiver spends in connecting, open and closed states.
  • The round trip time of alive messsageEvents.
  • No of crEvents transmitted to/from partner clients.
  • The current state/substate of the crSocket and it's internal eventReceiver and eventSender objects.
  • Conficuration parameters assigned to the crSocket: clientId, accessToken and partnerClientId.

It is also possible to monitor state changes and internal events in the JavaScript console.
